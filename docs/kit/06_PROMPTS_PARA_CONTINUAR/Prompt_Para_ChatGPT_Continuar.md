# PROMPT PARA CONTINUAR COM CHATGPT

Leia este material antes de qualquer resposta.

Estamos trabalhando no projeto Direciona Corretor, um copiloto comercial para corretores de imoveis focado em conducao de atendimento via WhatsApp.

O Direciona nao e CRM tradicional. Ele deve transformar conversas reais do WhatsApp em acao comercial pratica.

Promessa central:
O corretor abre o Direciona e entende quem atender, por que atender, quando atender e o que falar.

Regra-mae:
Inteligente por tras. Simples na frente.

Fluxo que nao pode falhar:
WhatsApp -> ZIP -> TXT + audios -> transcricao -> timeline cronologica -> analise de perfil/objecoes/timing -> prioridade -> proxima acao -> 3 mensagens -> salvar lead -> acompanhamento.

Nao remova funcao existente sem autorizacao.
Nao aplique material antigo inteiro sem analisar.
Nao transforme o sistema em CRM pesado.
Nao trate build como prova de funcionamento.
Nao exponha erro tecnico cru ao usuario.
Priorize sempre o nucleo: ZIP, audio, timeline, analise, mensagem e lead salvo.

Ao continuar, siga esta ordem:
1. Entender o estado atual.
2. Verificar se o motor principal funciona.
3. Corrigir o que impede ZIP/audio/timeline/analise.
4. Preservar visual preto/grafite/neon.
5. Manter home simples: quem atender, por que, quando e o que falar.
6. So depois evoluir sofisticacao, login, planos e comercializacao.
